Integration Data Flow (Step by Step)
Buyer App
Buyer registers sellers
Buyer opens milk-buyer.html, adds sellers with a unique Seller ID (e.g. SEL-001), name, and phone. Seller data is saved to buyerSellers.
// Buyer localStorage
buyerSellers → [{ id: "SEL-001", name: "Rajan Dairy", ... }]
Buyer App → Shared Layer
Buyer publishes to shared storage
On every entry, payment, or when "Live Sync" is clicked, the buyer writes a full snapshot to the shared integration key: milkSharedData. This includes sellers list, all entries, all payments, price, and unit.
// Integration key — readable by both apps
milkSharedData → {
  sellers: [...],
  entries: { "SEL-001": [...] },
  payments: { "SEL-001": [...] },
  price: 60,
  unit: "L",
  updatedAt: 1700000000000
}
Seller App
Seller sets up their profile
Seller opens milk-seller.html for the first time. A setup modal prompts for their name and Seller ID. The ID must match what the buyer registered. Profile is stored in sellerProfile.
// Seller localStorage
sellerProfile → { id: "SEL-001", name: "Rajan Dairy", ... }
Shared Layer → Seller App
Seller reads buyer's price & unit
On load and every 5 seconds, the seller reads milkSharedData. If the seller ID matches a buyer-registered seller, it auto-pulls the buyer's price, unit, and existing entries — ensuring both apps stay in sync.
// Seller reads from shared layer
const shared = JSON.parse(localStorage.getItem('milkSharedData'));
const found = shared.sellers.find(s => s.id === profile.id);
// Pull price & unit automatically
priceInput.value = shared.price;  // ← ₹60
currentUnit = shared.unit;      // ← "L"
Seller App
Seller logs deliveries
Seller adds morning/evening milk quantity entries. These are saved to sellerEntries locally and pushed back into the shared layer under their seller ID key — making it instantly available in the buyer's app.
// Seller writes back to shared data
shared.entries["SEL-001"] = sellerEntries;
shared.payments["SEL-001"] = sellerPayments;
localStorage.setItem('milkSharedData', JSON.stringify(shared));
Buyer App
Buyer sees updated data
When buyer clicks "Live Sync with Sellers" or adds entries, the buyer app reads from milkSharedData and refreshes all views — reflecting both buyer-entered and seller-entered data in the unified dashboard.

Quick Start Guide
How to connect Buyer ↔ Seller (Same Browser)
1
Open milk-buyer.html — set your price (₹/unit) and unit (L or kg).
2
Click "Add New Seller" and enter seller details. Use a memorable ID like SEL-001. This publishes data to milkSharedData.
3
Open milk-seller.html in another browser tab. Enter your name and the exact same Seller ID used in the buyer app.
4
The seller app will say "Buyer connected · ID matched" — price and unit auto-sync from the buyer.
5
Both apps can now add entries independently. Seller entries flow back into the buyer dashboard via the shared layer every time either app saves data.
6
In the buyer app, click "Live Sync with Sellers" anytime to refresh and pull the latest seller data into the dashboard.
